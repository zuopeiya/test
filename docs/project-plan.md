# 个人名片网站项目规划

## 1. 项目目标

打造一个前后端分离的多人个人名片网站，用于展示个人定位、联系方式、作品入口、能力标签和可合作事项。访客可以创建自己的名片，并通过公开地址分享给任何人访问。

## 2. 当前版本范围

- 前端：首页、创建页、编辑页、公开名片页，适配桌面和手机。
- 后端：Node.js 原生 HTTP API，提供多名片创建、读取、更新和健康检查接口。
- 数据：使用 JSON 文件保存多张名片，便于快速修改和迁移。
- 运行：前端和后端分别启动，端口分别为 `5173` 和 `4000`。

## 3. 技术选型

| 模块 | 当前选型 | 原因 |
| --- | --- | --- |
| 前端 | HTML + CSS + JavaScript | 无需安装依赖，方便本地直接运行和理解 |
| 后端 | Node.js 原生 HTTP 服务 | 轻量、可移植，后续可平滑迁移到 Express 或 NestJS |
| 数据 | JSON 文件 | 当前资料规模小，维护成本低 |
| 部署 | 静态前端 + Node API | 符合前后端分离部署方式 |

## 5. 接口规划

### GET /api/profile

返回完整名片资料，包括基础信息、联系方式、外部链接、技能、服务和统计数据。

### GET /api/health

返回后端服务状态，用于本地检查和部署健康探测。

### 后续接口建议

- `POST /api/messages`：提交访客留言。
- `GET /api/messages`：管理端查看留言。
- `PUT /api/profile`：管理端更新名片资料。
- `POST /api/auth/login`：管理端登录。

## 6. 迭代路线

### 第一期：展示型名片

- 完成前后端目录拆分。
- 完成个人资料 API。
- 完成响应式名片首页。
- 完成基础文档和本地运行说明。

### 第二期：可管理名片

- 增加登录保护。
- 增加资料编辑页面。
- 增加头像和二维码上传。
- 增加字段校验和保存提示。

### 第三期：线索收集

- 增加合作意向表单。
- 增加邮件或企业微信通知。
- 增加留言列表和处理状态。
- 增加防刷限制。

### 第四期：正式部署

- 前端部署到静态托管平台。
- 后端部署到 Node.js 运行环境。
- 增加域名、HTTPS、访问统计和错误日志。
- 增加备份策略。

## 7. 验收标准

- 前端页面能在桌面和手机宽度下正常浏览。
- 前端资料来自后端 `GET /api/profile`。
- 后端 `GET /api/health` 返回可用状态。
- 断开后端时，前端能显示备用内容和提示。
- 项目根目录包含清晰的运行说明和规划文档。
